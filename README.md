README:
You are tasked with building a browser-based Flashcards Study App in 2 hours. You know HTML/CSS/JS, but some features (modals, animations, keyboard accessibility, data persistence) may be time-consuming. You will leverage AI coding assistants (GitHub Copilot, Windsurf, Cursor) to move faster while actively reviewing and refining AI output. AI can be inconsistent, so you must understand and fix the code it generates.



Reflection:
1. Where AI saved time: Almost everywhere. It took less than 3 mins getting a functioning flashcards application. Each functionality took an additional 3 minutes with basically consisted of writing a well-crafted prompt. As learners, I believe we are faster with the English language than we are with JS/TS languages so naturally AI would save us time.


2. At least one AI bug you identified and how you fixed it: I tried to add a mode called "Answer mode" where it would show the user the answer card first and it would flip to the question card afterwards. It was probably not a very way of learning therefore I had to explain the concept with a short paragraph and it was able to deliver.


3. A code snippet you refactored for clarity: No refactoring was done since the application is not fully complete/tested at this time.


4. One accessibility improvement you added: It automatically added code required for accessibility so I did not have room to make any accessibility improvement manually.


5. What prompt changes improved AI output: By writing prompts that did not already assume knowledge of a certain functionality and therefore breaking down the desired functionality to be simple and asking it to turn to code.
